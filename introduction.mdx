## How to Get Started

Follow these steps to launch your first documentation site.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Sign up at [easyshoppingclub.com](https://easyshoppingclub.com) with your email or GitHub.
  </Step>
  <Step title="New Workspace" icon="plus">
    Click "New Workspace" and name it for your project, like `my-app-docs`.
  </Step>
  <Step title="Upload Docs" icon="upload">
    Drag and drop your MDX files or connect your Git repo.
  </Step>
  <Step title="Publish" icon="globe">
    Preview, then hit Publish. Your site goes live at a custom URL.
  </Step>
</Steps>

### Platform-Specific Setup

<Tabs>
  <Tab title="npm (Node.js)" icon="package">
    Install the CLI for local development:

    <CodeGroup tabs="CLI,SDK">
      ```bash
      npm install -g @easyshoppingclub/cli
      easydoc init my-project
      easydoc dev
      ```
      ```javascript
      // SDK example for embedding
      import { EasyDoc } from '@easyshoppingclub/sdk';
      const doc = new EasyDoc({ apiKey: 'your-key' });
      await doc.publish('/introduction');
      ```
    </CodeGroup>
  </Tab>
  <Tab title="GitHub" icon="github">
    Connect your repo directly:

    1. Go to Integrations > GitHub.
    2. Authorize and select repo.
    3. Docs auto-build on push.
  </Tab>
  <Tab title="VS Code" icon="code">
    Use the extension:

    ```bash
    code --install-extension easyshoppingclub.vscode-docs
    ```

    Edit MDX with live preview and one-click publish.
  </Tab>
</Tabs>

<Expandable title="Advanced Configuration" default-open="false">
  Customize your site with a `easydoc.config.js`:

  ```javascript
  module.exports = {
    theme: 'default',
    nav: [
      { title: 'Intro', href: '/introduction' },
      { title: 'API', href: '/api' }
    ],
    sidebar: true
  };
  ```

  Deploy with `easydoc deploy`.
</Expandable>
